# Release 2026.03.3

## Highlights

Product rebrand: **Aplikasi Akunting** is now **Balaka**. New name, logo, favicon, domain, and GitHub repo.

Four new feature phases: PPh 21 TER method, Scheduled Payroll, Free-Form Journal Entry API, and SPT Tahunan Badan Data Export.

Six tax filing improvements (FR-001–006) based on actual SPT Tahunan Badan 2025 filing via Coretax. New PDF export and Coretax-compatible JSON endpoint.

No schema migration changes. No breaking API changes.

## What's New

### Phase 17: SPT Tahunan Badan Data Export

- L1 Rekonsiliasi Fiskal (P&L, koreksi fiskal, kompensasi rugi, PKP, PPh Badan)
- L4 Penghasilan Final (PPh 4(2) per kode objek pajak)
- L9 Penyusutan & Amortisasi (DJP Converter DATA+REF format)
- Transkrip 8A Laporan Keuangan (Neraca + Laba Rugi)
- BPA1 e-Bupot PPh 21 1721-A1 (all employees, DJP Converter format)
- Consolidated lampiran endpoint with Coretax field number mapping
- Fiscal loss carryforward management (5-year FIFO per UU PPh Pasal 6 ayat 2)
- SPT Checklist dashboard with 7 data readiness checks
- All endpoints support JSON (default) and Excel (`?format=excel`)

### Phase 18: PPh 21 TER Method (PMK 168/2023)

- TER (Tarif Efektif Rata-rata) calculation for monthly PPh 21 (Jan–Nov)
- December reconciliation using progressive annual rates
- TerCategory enum mapped from PTKP status
- Web calculator for PPh 21 TER simulation

### Phase 19: Scheduled Payroll

- `PayrollSchedule` entity for automated monthly payroll run creation
- Single-row config: dayOfMonth, baseSalary, jkkRiskClass, autoCalculate, autoApprove, active
- CRUD API: `GET/POST/DELETE /api/payroll/schedule`
- Daily scheduler (6:30 AM cron) with startup catch-up for missed runs
- Never auto-posts — always requires manual posting step

### Phase 20: Free-Form Journal Entry API

- `POST /api/transactions/journal-entry` — arbitrary debit/credit lines without template
- For closing entries, adjusting entries, and opening balances

### Tax Filing Improvements (FR-001–006)

- **FR-001: L1 non-operating expenses** — `otherExpenses` now captures all non-5.1 expenses (5.2 Beban Lain, 5.3 Beban Zakat, 5.9 PPh/Denda, etc.) via exclusion filter
- **FR-002: L9 depreciation fallback** — Pool assets with no individual depreciation entries now calculate from monthly formula × months active in year
- **FR-003: PKP rounding** — PKP rounded down to nearest 1,000 before PPh Badan calculation (UU PPh pasal 6 ayat 3). Both `pkp` and `pkpRounded` returned in API
- **FR-004: excludeClosing parameter** — `GET /api/analysis/income-statement` now accepts `excludeClosing=true` to exclude closing journal entries
- **FR-005: Financial statements PDF** — `GET /api/tax-export/financial-statements/pdf?year=` generates a 2-page PDF (Neraca + Laba Rugi) with company name/NPWP header for Coretax upload
- **FR-006: Coretax SPT export** — `GET /api/tax-export/coretax/spt-badan?year=` returns structured JSON matching Coretax form fields with plain numbers for direct data entry

## Migration Guide

### For existing deployments

1. Update git remote on server: `git remote set-url origin https://github.com/artivisi/balaka.git`
2. Update DNS: add A record for new domain pointing to the same server IP
3. Deploy with Ansible to set up nginx vhost + SSL for new domain
4. Add nginx redirect from old domain to new domain
5. Update Telegram webhook URL if bot is enabled

### Theme override

If your deployment overrides `app.theme.name` via Ansible vars (e.g., `app_theme_name: artivisi`), no change needed — client themes are unaffected. The new `balaka` default only applies when no override is set.
