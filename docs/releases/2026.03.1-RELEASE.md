## What's New

### Phase 14: Fiscal Adjustments API

- CRUD endpoints: `GET/POST/PUT/DELETE /api/fiscal-adjustments`
- Filter by tax year: `?year=2025`
- Categories: PERMANENT, TEMPORARY; Directions: POSITIVE, NEGATIVE
- Integrates with existing `GET /api/tax-export/rekonsiliasi-fiskal`

### Phase 15: Payroll API + PPh 21 / 1721-A1

- Employee CRUD: `GET/POST/PUT /api/employees` with salary component assignment
- Salary component CRUD: `GET/POST/PUT/DELETE /api/salary-components`
- Payroll run lifecycle: create ‚Üí populate ‚Üí calculate ‚Üí approve ‚Üí post
- PPh 21 calculation using TER method (PMK 168/2023) with December annual reconciliation
- 1721-A1 generation: `GET /api/employees/{id}/1721-a1?year=YYYY`
- Annual PPh 21 summary: `GET /api/pph21/summary?year=YYYY`

### Phase 16: User Manual Revamp

- Renamed section: "Bantuan AI untuk Pencatatan Transaksi" ‚Üí "Operasi Aplikasi dengan Bantuan AI"
- Covers full lifecycle: installation, data analysis, COA/template generation, seed data import, transaction posting, payroll, vendor bills, bank reconciliation, tax filing
- API reference updated for all 16 controllers

### OpenAPI Documentation Updates

- Clarified DRAFT workflow: DRAFTs have no journal entries until posted, use `journal-preview` endpoint
- Added workflows: Fiscal Adjustments, Payroll Processing, PPh 21 Annual / 1721-A1
