# Release 2026.02.3-RELEASE

## Highlights

- **Phase 10 — Invoice & Bill Management**: Full invoice lifecycle, vendor bills with REST API, payment tracking, aging reports, customer/vendor statements
- **Phase 11 — Recurring Transactions**: Template-based scheduling with auto-posting for repeating transactions
- **Smart Alerts**: Configurable alert rules with dashboard widget and daily evaluation
- **Transaction Tags**: Tag types, tag CRUD, transaction assignment, filtering, and reporting

## What's New

### Invoice & Bill Management (Phase 10)

- Invoice line items with tax calculation
- Vendor entity and vendor bill management
- `POST /api/bills` — REST API for AI-assisted bill entry
- Payment tracking with partial payment support
- Aging reports (receivable and payable)
- Customer and vendor statements

### Recurring Transactions (Phase 11)

- Recurring transaction templates with configurable schedule (daily, weekly, monthly, yearly)
- Auto-posting via scheduler
- Manual trigger and skip controls
- Dashboard showing upcoming and overdue recurring transactions

### Smart Alerts (Phase 9.2)

- 7 evaluator types: cash low, receivable overdue, expense spike, project cost overrun, project margin drop, collection slowdown, client concentration
- Configurable alert rules with severity levels
- Daily evaluation at 8am via cron
- 24h deduplication for unacknowledged events
- Dashboard widget and dedicated alert list page

### Transaction Tags (Phase 9.3)

- Tag type and tag CRUD management
- Tag assignment to transactions
- Filtering transactions by tag
- Tag-based reporting

### Other Improvements

- Self-service device token management UI at /settings/devices
- API pagination for analysis reports and drafts endpoints
- Static /api/capabilities.json for AI assistant discovery
- User manual sections for alerts (section 15) and invoices/bills

## Bug Fixes

- Fixed flaky TagCrudTest.shouldCreateNewTag timeout in CI
- Fixed PaymentTrackingTest and StatementTest resilience in full suite
- Resolved GitHub code scanning alerts
- Upgraded minimatch to resolve ReDoS vulnerability (CVE-2026-26996)
- Resolved SonarCloud reliability issues in HTML templates

## Migration Guide

New database migrations required: V010 (payment tracking), V011 (recurring transactions).

Standard upgrade process:

1. Pull latest code
2. Run `./mvnw clean package`
3. Deploy using Ansible or remote-build script
4. Flyway will auto-apply V010 and V011 on startup
