# Release 2026.02.5-RELEASE

## Highlights

- **Direct Draft Creation**: New `POST /api/drafts` endpoint creates DRAFT transactions directly with template + account overrides, bypassing the from-text/from-receipt workflow.
- **Journal Preview**: New `GET /api/transactions/{id}/journal-preview` endpoint previews journal entries before posting.
- **Template Lines in API**: `GET /api/templates` now returns template lines with account details and accountHint.
- **409 Conflict Fix**: `PUT /api/transactions/{id}` with `lineAccountOverrides` no longer returns 409 on repeated calls.

## What's New

### New Endpoints

| Method | Path | Description |
|--------|------|-------------|
| POST | `/api/drafts` | Create DRAFT transaction directly with templateId + lineAccountOverrides |
| GET | `/api/transactions/{id}/journal-preview` | Preview journal entries for a DRAFT transaction |

### API Response Improvements

- **Template lines exposed**: `GET /api/templates` and `GET /api/templates/{id}` now include a `lines` array with `lineOrder`, `position`, `accountId`, `accountCode`, `accountName`, `accountHint`, `formula`, `description`.
- **transactionId in DraftResponse**: `POST /api/drafts/{id}/approve` now returns `transactionId` (UUID of created Transaction) so API consumers can proceed to PUT/POST without querying.
- **draftId naming**: Documented that draft endpoints return `draftId` (not `id`) to distinguish draft from transaction context.

### Bug Fixes

- **409 Conflict on re-apply lineAccountOverrides**: `replaceAccountMappings()` now flushes the DELETE before INSERT to avoid unique constraint violation when the same overrides are sent twice.
- CI timeout bumped to 60min, CodeQL upgraded to v4.
- SonarCloud code smells resolved (S6814, S6809, S1192, S3776, S2184).

### Documentation

- User manual (`13-bantuan-ai.md`) updated with all new endpoints, template lines example, journal preview section, and updated API reference table.
- `capabilities.json` updated with new endpoints and "Draft Then Post" workflow.

None. All changes are additive.
